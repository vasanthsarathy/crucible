Friction points and improvement ideas captured while **using** `crucible-rdd` on
real research projects — the raw material for issues and future versions.

**How this gets filled:** jot fast the moment something bugs you (or use the
personal `/friction` command from any research session). Don't fix mid-research.
In batches, triage rows into GitHub issues (bug/feature templates), then delete
the triaged rows.

**Severity:** 1 = minor nit · 2 = mild friction · 3 = actively got in the way.
